Claude: Excellent catch! This is **critical to understand** because `<` and `>` have **different meanings** in different places.

---

## **`<` and `>` = Redirection (outside if statements)**

- `<` = **input redirection** ("take data FROM a file")
- `>` = **output redirection** ("send data TO a file")
- `>>` = **append** ("add to end of file")

Examples:
```bash
wc -c < file.txt          # Read FROM file.txt
echo "text" > file.txt    # Write TO file.txt (overwrites)
echo "text" >> file.txt   # Add TO file.txt (appends)
```

---

## **`-gt`, `-lt`, `-eq` = Comparison (inside if statements)**

These are used ONLY inside `[ ]` in if statements:

- `-gt` = **greater than** (>)
- `-lt` = **less than** (<)
- `-eq` = **equals** (=)
- `-ge` = **greater or equal** (>=)
- `-le` = **less or equal** (<=)

Examples:
```bash
if [ $count -gt 10 ]; then      # if count > 10
if [ $count -lt 10 ]; then      # if count < 10
if [ $count -eq 10 ]; then      # if count = 10
```

---

## **Why two different symbols?**

In bash:
- `<` and `>` are used for **file redirection**
- `-gt`, `-lt` are used for **number comparison in if statements**

If you tried `if [ $count > 10 ]` it would be interpreted as "redirect to file 10" (wrong!).
